# Danh sách bảng Report DB — Sync từ ERP

> **Tài liệu:** Blueprint Section 5.3 — Controlled vocabularies  
> **Phase:** 1 (ưu tiên đầu tiên)  
> **Cập nhật:** 2026-07-17 (đã xác nhận tên bảng ERP nguồn)  
> **Liên quan:** `2026-07-18-Implementation-Plan.md`, `init-docs/2026-07-09_ProjectAI_erp-database-build-blueprint.pdf`

---

## Mục tiêu Phase 1

Đồng bộ **toàn bộ bảng danh mục mã chuẩn (5.3)** từ SQL Server ERP sang PostgreSQL Report DB.

Đây là nền tảng FK cho Phase 2 (5.2 reference/rate-card) và Phase 3 (5.1 operational). **Phải hoàn thành Phase 1 trước** khi sync các nhóm còn lại.

### Đặc điểm chung nhóm 5.3

| Tiêu chí | Giá trị |
|----------|---------|
| Nguồn dữ liệu | ERP (authoritative) |
| Độ phức tạp transform | Thấp — chọn cột, rename, chuẩn type |
| Sync mode đề xuất | **Full-load** (bảng nhỏ, ít thay đổi); có thể bật CDC sau nếu cần nhất quán |
| Tần suất đề xuất | 1 lần/giờ hoặc 1 lần/ngày (tùy SLA report) |
| Quy tắc blueprint | `code` là key, `name` là label; cột mã ERP dùng prefix `ua_` |
| Convention đích | `lower_snake_case`, `timestamptz` cho timestamp |

### Cột metadata chung (mọi bảng Phase 1)

Mỗi bảng report nên có thêm cột phục vụ sync (không có trong ERP):

| Cột | Kiểu | Mục đích |
|-----|------|----------|
| `synced_at` | `timestamptz` | Thời điểm worker ghi lần cuối |
| `source_system` | `text` | Mặc định `'erp'` — dự phòng nguồn khác sau này |

### Thứ tự sync trong Phase 1

Sync theo thứ tự dưới đây để tránh lỗi FK nội bộ phase:

```
1. units
2. unit_conversions          (FK → units)
3. cmp_sections
4. fabric_kinds
5. fabric_types              (FK → fabric_kinds, nếu có)
6. trim_groups
7. trim_types                (FK → trim_groups, nếu có)
8. treatment_types
9. work_types
10. style_categories
11. garment_kinds
12. colours
13. seasons
14. artwork_positions
15. machines
16. exchange_rates
17. drop_vocab               (FK → seasons, nếu có; nguồn: ERP.Configs.Drops)
```

> **Lưu ý:** Tên bảng ERP SQL Server đã được **xác nhận** (xem từng mục). Cột ERP vẫn cần verify bằng `INFORMATION_SCHEMA` hoặc script introspection trên DB thật trước khi code mapper. Cột **UA reference file** lấy từ export catalog UA (bằng chứng nghiệp vụ, không phải tên bảng SQL).

## Bảng 01 — `units`

### Mục tiêu

Danh mục **đơn vị tính** chuẩn ERP (KG, M, ROLL, PCS, SET, CONE, YARD…). Mọi cột `ua_unit_code` trong BOM, purchase, inventory đều FK về đây.

### Schema Report DB

| Cột | Kiểu | PK/FK | Bắt buộc | Mô tả |
|-----|------|-------|----------|-------|
| `ua_unit_code` | `text` | **PK** | ✓ | Mã đơn vị ERP, vd. `PCS`, `KG`, `M` |
| `name` | `text` | | ✓ | Tên hiển thị, vd. `Pieces`, `Kilogram` |
| `description` | `text` | | | Mô tả thêm (nếu ERP có) |
| `active` | `boolean` | | ✓ | `true` = đang dùng; lọc record inactive khi sync |
| `synced_at` | `timestamptz` | | ✓ | Metadata sync |
| `source_system` | `text` | | ✓ | Mặc định `'erp'` |

**Ước lượng:** ~14 dòng

### Nguồn ERP

| Hạng mục | Giá trị |
|----------|---------|
| Bảng ERP | **`ERP.Configs.Units`** |
| UA reference | `ERP CATALOG/Master Data/` (unit of measure) |
| Điều kiện lọc | Chỉ lấy record `active = 1` / không bị soft-delete |

### Mapping ERP → Report (dự kiến)

| ERP column (TBD — verify schema) | Report column | Ghi chú |
|----------------------------------|---------------|---------|
| `UnitCode` / `UOM_CODE` | `ua_unit_code` | Uppercase, trim |
| `UnitName` / `UOM_NAME` | `name` | |
| `Description` | `description` | Optional |
| `IsActive` / `Status` | `active` | Map 1/0 hoặc Y/N |

## Bảng 02 — `unit_conversions`

### Mục tiêu

Quy đổi giữa các đơn vị (vd. M ↔ YARD). Dùng khi tính consumption hoặc hiển thị đa đơn vị.

### Schema Report DB

| Cột | Kiểu | PK/FK | Bắt buộc | Mô tả |
|-----|------|-------|----------|-------|
| `from_unit_code` | `text` | **PK**, FK → `units` | ✓ | Đơn vị nguồn |
| `to_unit_code` | `text` | **PK**, FK → `units` | ✓ | Đơn vị đích |
| `conversion_factor` | `numeric(18,8)` | | ✓ | Hệ số: `1 from = factor × to` |
| `active` | `boolean` | | ✓ | |
| `synced_at` | `timestamptz` | | ✓ | |
| `source_system` | `text` | | ✓ | |

**Ước lượng:** ~2 dòng (theo blueprint)

> **Sync contract:** Đây là composite PK (`from_unit_code`, `to_unit_code`). Writer phải build cả `WHERE` delete và `ON CONFLICT` upsert từ **đủ hai cột PK**, không được giả định single-column key.

### Nguồn ERP

| Hạng mục | Giá trị |
|----------|---------|
| Bảng ERP | **`ERP.Configs.UnitRates`** |
| UA reference | `ERP CATALOG/Master Data/` |
| Ghi chú | Tên ERP là `UnitRates` — map sang report `unit_conversions` (cùng nghiệp vụ quy đổi đơn vị) |

### Mapping ERP → Report (dự kiến)

|-----|------|-------|----------|-------|
| `ua_section_code` | `text` | **PK** | ✓ | Mã section: `C`, `N`, `P`, `S`, `B`, `E`, `P1`, `F`, `P2` |
| `name` | `text` | | ✓ | Tên đầy đủ, vd. `Cutting`, `Sewing` |
| `sort_order` | `smallint` | | | Thứ tự hiển thị trên dây chuyền |
| `active` | `boolean` | | ✓ | |
| `synced_at` | `timestamptz` | | ✓ | |
| `source_system` | `text` | | ✓ | |

**Ước lượng:** 9 dòng (cố định)

### Nguồn ERP

| Hạng mục | Giá trị |
|----------|---------|
| Bảng ERP | **`ERP.Configs.CMP.Operations`** |
| UA reference | `ERP CATALOG/Technical Catalog/managecmpoperation.xls` |
| Ghi chú | Nguồn là bảng **Operations**. Cần kiểm tra schema: nếu mỗi row là 1 operation thuộc 1 section → reader/mapper SQL Server phải `SELECT DISTINCT` hoặc `GROUP BY section_code` **trước** upsert để ra 9 section; nếu bảng đã chứa sẵn danh mục section thì sync trực tiếp. Orphan-delete phải so tập PK sau aggregate, không phải raw operation rows. Chi tiết operation timing để ở Phase 2 (`cmp_timing`). |

### Mapping ERP → Report (dự kiến)
